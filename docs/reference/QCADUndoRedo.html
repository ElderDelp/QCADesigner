Undo/Redo

QCADesigner's Undo/Redo implementation is based on the QCADUndoEntry class. QCADUndoEntryGroup provides an Undo/Redo stack and qcad_undo_entry_group_get_default() allows you to retrieve an application-wide Undo/Redo stack you can push QCADUndoEntry objects onto. The actual Undo/Redo work can be performed one of two ways:

  • by subclassing QCADUndoEntry and overriding its fire() virtual function,
  • or by connecting to its "apply" signal.

The qcad_undo_entry_signal_connect() convenience function will connect your handler to the signal and will ensure that data will be freed when the object is destroyed.
